# Core Web Vitals

Use these as the default performance baseline.

- LCP: loading experience
- INP: interaction responsiveness
- CLS: visual stability

Practical targets used by web.dev:
- LCP: good at or below 2.5s
- INP: good at or below 200ms
- CLS: good at or below 0.1

Guidance:
- Optimize the largest content element first for LCP.
- Reduce long tasks and expensive event handlers for INP.
- Reserve space and avoid late layout changes for CLS.
