# Vue I18n

## Setup

Use Vue I18n Composition API mode for Vue 3:

```ts
import { createI18n } from 'vue-i18n'

export const i18n = createI18n({
  legacy: false,
  locale: 'en',
  fallbackLocale: 'en',
  messages: {
    en: {},
  },
})
```

In components, use `useI18n()`:

```vue
<script setup lang="ts">
import { useI18n } from 'vue-i18n'

const { t, n, d } = useI18n()
</script>
```

## Translation Rules

- Keep keys stable and domain-oriented, for example `checkout.payment.submit`.
- Use interpolation for dynamic values, not string concatenation.
- Use pluralization APIs for counts.
- Use number and datetime formatters for locale-aware values.
- Avoid putting translated strings in shared composables unless they own copy behavior.
- Keep fallback locale behavior explicit and tested.

## Locale Loading

- Split large message catalogs by locale and feature when bundle size matters.
- Load the active locale before rendering text-heavy routes when flicker would be visible.
- Update `document.documentElement.lang` and `dir` when locale changes.
- Coordinate translated route titles with route metadata and SEO metadata.

## Component Scope

Use global scope for app-wide UI copy. Use local component messages only when a component is packaged independently or owns highly specific copy that should not leak into the app catalog.
