# Contributing to Web UI Skills

Thanks for your interest in contributing. This document highlights the main practices.

1. Fork the repo and open a feature branch.
2. Keep changes small and focused; one logical change per PR.
3. Follow existing SKILL.md structure for new skills.
4. Add tests or examples where applicable.
5. Run `npm run lint:md` to validate Markdown.
6. Update `README.md` and bump `package.json` version for releases.

For full contribution workflow see `AGENTS.md`.
